Unit 2: Object Oriented Analysis

Learning Outcomes:

In this unit, we shall:

  • Explore the ways in which actors interact with software in terms of functional requirements of a piece of software
  • Identify the interactions and behaviours invoked within software once active using a state machine diagram

On completion of this unit, you will be able to:

  • Design a use case diagram for a software development scenario
  • Create a state machine diagram to examine the behaviours and interactions within a system

Notes

State Machines: A model that describes the behaviour of an object in response to events it receives.

Events: A noteworthy occurence that is characterized by its consequences.

Call Event: A reception of a call by an object that implements an operation as a state machine transition instead of a fixed procedure.

Change Event: The satisfaction of a boolean expression that depends on specific attribute values.

Time Event: Caused by signals from the operating system or an object in the application - an event representing the passage of time.

State: Represents a period of time in an objects life.

Transition: Represents the response of an object to an event - consists of a trigger, a guard condition, an action, and a target state.

Trigger Event: The trigger of a transition is an event that allows the transition to occur.

Guard Condition: Condition that is evaluated when a trigger event occurs to determine if the transition fires.

Completion Transition: Triggered by the completion of an activity in the state it leaves.

Action: A brief computation executed when a transition fires. Can be an assignment statement, arithmetic computation, sending a signal to another object, etc.

Change of State: When the execution of the action is complete, the target state of the transition becomes active.

Nested States: States can be nested within other composite states.

Entry and Exit Actions: A transition can cause exit and entry of states. A state may have actions that are performed when it is entered or exited.

Internal Transition: Has a source but no target state, meaning it does not change the active state when fired.

Composite State: Decomposed into sequential or concurrent substates - entering or exeting the composite state will trigger the corresponding entry/exit actions.

Hashmap: Also known as dictionary or associative array - data structure that allows you to store and retrieve values based on a key.

Polymorphic Functions: Functions that work with several types.

Debugging

There are several ways to check if values/arguments are valid.

Example function:

def valid_time(time):
if time.hour < 0 or time.minute < 0 or time.second < 0:
return False
if time.minute >= 60 or time.second >= 60:
return False
return True

We can use this function to check our code. For example:

def add_time(t1, t2):
if not valid_time(t1) or not valid_time(t2):
raise ValueError('invalid Time object in add_time')
seconds = time_to_int(t1) + time_to_int(t2)
return int_to_time(seconds)

We can also use assert statements:

def add_time(t1, t2):
assert valid_time(t1) and valid_time(t2)
seconds = time_to_int(t1) + time_to_int(t2)
return int_to_time(seconds)

For debugging purposes, keep this function handy:

 
def print_attributes(obj);								
	for attr in vars(obj);
		print(attr, getattr(obj, attr));

Further experimentation with protected and unprotected values:


class ProtectedValue:
	def __init__(self, value):
		self.value = value

	def get_value(self):
		return self.value

	def set_value(self, value):
		self.value = value


class UnprotectedValue:
	def __init__(self, value):
		self.value = value

p = ProtectedValue(10)
print(p.get_value()) # Output: 10

u = UnprotectedValue(20)
print(u.value) # Output: 20

In this program, the ProtectedValue class represents a protected value that can only be accessed using its get_value method and modified using its set_value method. On the other hand, the UnprotectedValue class represents an unprotected value that can be accessed and modified directly. This distinction is useful in scenarios where you want to ensure that the value is only manipulated in a certain way or you want to encapsulate the data to prevent direct manipulation.
